Объектная модель (Hierarchy)
В Roblox все, что ты видишь в игре – это объекты (Parts, Scripts, Sounds). Они организованы в иерархию "Отец - Ребенок" (Parent - Child).

Workspace – главный контейнер для всех физических объектов.

Script – мозг объекта. Чтобы изменить объект, скрипт должен знать, где он находится по отношению к нему.

script.Parent – ​​обращение к объекту, внутри которого лежит скрипт

Свойства — это характеристики объекта, которые мы изменяем через код. Это форматы, в которых мы передаем значение свойствам:

Number/Integer: Числа (1, 0.5, -10).

String: Текст в кавычках ("Hello", "Gold").

Boolean: Логическое значение (true или false).

Vector3: Набор трех чисел для 3D пространства (Vector3.new(x, y, z)).

Color3: Цвет RGB (Color3.fromRGB(r, g, b) — от 0 до 255).

4. Схема работы скрипта (Алгоритм)
Чтобы изменить свойство, мы проходим три шага:

Путь: Найти объект (local myPart=script.Parent).

Сменная: Сохранить путь в переменную (для удобства).

Изменение: Вызвать свойство через точку и присвоить значение.
